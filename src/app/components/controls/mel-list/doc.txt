
Statusdiagram
  Possible Status of the List and Columns:
  None  : Nothing selected
  RnV   : Row n is selected (View only)
  RnEc  : Row n Field c is selected (Editable)
  Actions:
  CnV   : Click on Row n (view -> not editable field)
  CnEc  : Click on Row n in an editable field in column c
  Cex   : Click outside the list

  Statustransitions: (List has mor than one rows)
  Status  Action  Status'   Click...
  ---------------------------------
  None    CnV     RnV       ... in a row
          CnEc    RnEc      ... in an editable field
          Cex     None      

  RnV     CnV     None      ...in the selected row
          CmV     RmV       ...in another row than n
          CnEc    RnEc      ...in an editable field
          Cex     RnV

  RnEc    CnV     RnV       ... in a not editable field in row n
          CmV     RmV       ... in a not editable field in another row than n
          CnEc    RnEc      ... in an editable field in the same row
          CmEc    RmEc      ... in an editable field in another row than n
          Cex     RnV        

  Click, Key-Navigation 
  case        Action        (currEditable)                                Note                  
  -------------------------------------------------------------------------------------------------------------------------------------
  nothing     click(td[1,1])  edit[1,1] rowClicked[1]                     nothing is selected an we click in an editable tablecell
  nothing     click(tr[x])              rowClicked[1]                     nothing is selected an we click in a not editable tablecell 
  nothing     click(other)                                                nothing is selected an we click outside the list
  edit[1,1]   click(td[1,1])      (1,1) rowClicked[1]                     click in the current editable field
  edit[1,1]   click(td[1,2])  edit(1,2) rowClicked[1] view[1,1]           click in an editable field in the same row
  edit[1,1]   click(td[2,3])  edit(2,3) rowClicked[2] view[1,1]   update  click in an editable field in another row
  edit[1,1]   click(tr[1])        (1,1) rowClicked[1] view[1,1]   update  click in a not editable tablecell in the same row
  edit[1,1]   click(tr[2])        (1,1) rowClicked[2] view[1,1]   update  click in a not editable tablecell in the same row
  edit[1,1]   click(other)        (1,1)               view[1,1]   update  click outside the list
  // Keynavigation ony, when an editfield is in editmode. 
  edit[1,1]   Arrow Up/-Down  edit(2,1)               view[1,1]   update  
  edit[1,1]   (Shift)Tab      edit(1,2)               view[1,1]
  edit[1,1]   Enter               (1,1)               view[1,1]   update

  The keys have to simulate clicks on the target-cells to get the cells editable. Not editable fields are skipped
  We ignore the Arrow-Up-Key in the row with index = 0
  The Arrow-Down-Key in the last row will trigger an "Insert", if insert is allowed
  The Tab-Key in the last row simulate a click in the first column of the next row (possibly an insert, if in the last row)
  The Shift-Tab-Key in the first column is ignored

  Changing in editmode, the current editComponent is saved in "currEditableComponent"
  Changing in viewmode, the current editComponent will be undefined if its the same as the viewmode-Component
  After that we update the record (if dirty), 
  if  ( (changing to viewmode) 
      AND (the currentComponent is in another row than the view-mode-component OR the currentComponent is the viewComponent)
